var let const
var keyword means that it is a variable and thus the value can be re-assigned. let keyword means that it can let it's value be changed within the block scope where it is declared. So it can be re-assigned in certain places. const keyword means that it is a constant and thus the value cannot be re-assigned.
variables declared with var are scoped to the immediate function body variables declared with let are block scoped which means the variables will have scope to the immediate enclosing block. variables declared with const are scoped to the immediate function body
variables declared with the var keyword are hoisted. Hoisting means that the variable can be accessed in their enclosing scope even before they are declared. variables declared with the let keyword are not hoisted. variables declared with the const keyword are not hoisted.

Regular Function Arrow Function
Regular functions can be created using function declarations or expressions.
Syntax-1: function name(parameter){ lines of code}
Syntax-2: let x = function name(parameter){ lines of code} 		Arrow functions are created using expressions.
Syntax: let x = (parameter) => {lines of code}
They are constructible and callable. They are only callable, not constructible.
Since regular functions are constructible, they can be called using the new keyword. Since arrow functions are not constructible, we will get a run-time error on trying to construct a non-constructible arrow functions using the new keyword.
Arguments objects are available in regular functions Arguments objects are not available in arrow functions.
Regular functions have their own this. Arrow functions do not have their own this.

Template literals, also informally known as Template strings, are literals delimited with backtick (`) characters, allowing for multi-line strings, for string interpolation with embedded expressions, and more. Along with having normal strings, template literals can also contain other parts called placeholders, which are embedded expressions delimited by a dollar sign and curly braces: ${expression}. Template string is used to write multiline strings with dynamic features. Using dynamic values in same line as string, template string has made it easier to write codes in minimum time. We had to add strings with another string or value using plus signs before template string came with its dynamic features.

1. .map(): Morph Array Piece-by-piece is the elaboration of map. It takes every element of an array, does something with the elements, then returns a new array with the new elements. map() is used only for arrays. After declaring an array, we have to use .map() and pass a function as parameter.

Syntax: const newArray = inputArray.map(function);

2. .forEach(): It takes the elements of an array and does something without returning anything. This process is outdated now.

Syntax: array.forEach(function);

3. .filter(): It takes the elements of an array and does a conditional check, then returns an array of elements those meet the condition.

Syntax: const newArray = inputArray.filter(function);

4. .find(): It takes the elements of an array and starts to check if any of the element meets the condition given in function, then returns the first match as a number.

Syntax: array.find(function);
